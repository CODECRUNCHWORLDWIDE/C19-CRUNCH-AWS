# Week 9 — Challenges

One challenge this week. It is harder and more open-ended than the exercises: there is no single right answer, only a defensible one with numbers behind it. Budget real time for it — the cost reasoning is the kind of thing you will do for a living.

| Challenge | File | What you do | Time |
|-----------|------|-------------|------|
| 1 | [challenge-01-on-demand-vs-provisioned-cost.md](./challenge-01-on-demand-vs-provisioned-cost.md) | Switch the single table from on-demand to provisioned with autoscaling. Measure the cost delta at steady-state, burst, and idle. Recommend the right mode for each load profile, with dollar figures. | 2–3h |

The challenge produces the on-demand-vs-provisioned cost report that the mini-project requires, so do it before you finish the mini-project. It compounds: the same reasoning recurs for Aurora in Week 8 (ACUs), for Lambda concurrency in Week 10, and for the whole capstone FinOps story in Week 14.
